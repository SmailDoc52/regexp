# Домашнее задание «Регулярные выражения» (Решение)

Репозиторий содержит решение задачи по приведению в порядок адресной книги с использованием регулярных выражений и инструментов декомпозиции кода.

## Особенности реализации

В данном решении применил знания из следующей лекции итераторы, генераторы, с акцентом на декомпозицию:

1. **Декомпозиция и ООП**: Всю логику работы с файлами и очистки номера вынес в отдельный класс `CSVManager` и вынес его в отдельный файл `tools/csv_manager.py`.
2. **Итераторы и Генераторы**: Функции чтения и записи файла написал «с нуля». Вместо полной загрузки файла в оперативную память `csv_reader` считывает и отдает строки построчно с помощью `yield` (из лекции генераторы). 
3. **Регулярные выражения**: Метод `cleaned_phone`, обрабатывающий телефонные номера, поместил в класс `CSVManager` так как метод обрабатывает номера из файла.

## Описание структуры и методов класса `CSVManager`

Класс разработан для инкапсуляции логики работы с файлами формата CSV и первичной нормализации данных. Контекст методов жестко разделен по принципу единственной ответственности (SRP).

### 1. Конструктор `__init__(self, file)`
* **Назначение**: Инициализирует объект менеджера, связывая его с конкретным путем к файлу.
* **Аргументы**: `file` (str) - путь к целевому CSV-файлу для чтения или записи.

### 2. Метод `csv_reader(self)`
* **Назначение**: Потоковое чтение данных из CSV-файла.
* **Реализация**: Вместо выгрузки всего файла в RAM, метод считывает и отдает строки поочередно с помощью `yield`. Встроен перехват исключения `StopIteration`

### 3. Метод `csv_writer(self, data_dict: dict, headers)`
* **Назначение**: Запись обработанного файла обратно в CSV-формат.
* **Аргументы**: 
  * `data_dict` (dict) - словарь, где ключом является кортеж уникальных имени и фамилии `(lastname, firstname)`, а значением - список остальных атрибутов контакта.
  * `headers` (list) - список заголовков столбцов таблицы.

### 4. Метод `cleaned_phone(phone)`
* **Назначение**: Стандартизация телефонных номеров в соответствии с ТЗ.
  * Приводит номер к шаблону `+7(XXX)XXX-XX-XX доб.XXXX`.
* **Тип метода**: `@staticmethod` (статический метод).
* **Аргументы**: `phone` (str) - сырая строка телефона из исходного файла.
* **Особенности**: 
  * Метод независим от состояния объекта (не использует данные экземпляра), поэтому объявлен как статический.
